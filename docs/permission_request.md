# Запрос разрешения на доступ и интеграции

Заказчик: Управление туризма Акмолинской области / КГУ «Smart Aqmola»
Проект: VisitAqmola — цифровая туристская платформа
Дата: 2025-11-28

## 1. Цель запроса
Получение разрешений и ключей доступа для интеграции платформы VisitAqmola с внешними и государственными сервисами, а также согласование условий обработки персональных данных.

## 2. Перечень требуемых доступов и интеграций
- eGov Mobile (OAuth2/OIDC):
  - Регистрация приложения, client_id / client_secret
  - Redirect URI: https://visit.aqmola.kz/auth/callback
  - Доступ к минимальному профилю: ФИО, ИИН (при необходимости), контактные данные
  - eGov Pay: мерчант-учётка, тестовый и боевой стенд
- Платёжные системы:
  - Kaspi QR: регистрация мерчанта, webhook URL, тестовые ключи
  - Банковские карты (эквайринг): договор, API-доступ, тестовые карты
- Туристические интеграции:
  - Freedom Travel API: доступ к каталогу туров/бронированию (sandbox, prod)
  - Booking.com API: партнёрский доступ к Search/Availability (при наличии)
  - TripAdvisor API: доступ к отзывам/рейтингам (если доступен партнёрский канал)
- Инфраструктурные ресурсы (по согласованию):
  - Домен(ы): visit.aqmola.kz, api.visit.aqmola.kz
  - DNS/SSL: выпуск и установка сертификатов
  - Object Storage (фото/видео/360°): доступ к бакетам

## 3. Объём персональных данных и обработка
- Минимально необходимые поля для аутентификации и бронирований (KYC по требованию).
- Правовые основания: согласие пользователя, договор-оферта, Положение о ПДн.
- Меры защиты: шифрование в транзите (HTTPS), в покое (KMS/at-rest), разграничение доступа, аудит.

## 4. Тестовые стенды и сроки
- Необходим тестовый доступ к eGov, платёжным и туринтеграциям в течение 10 рабочих дней.
- Предоставить контактное лицо для оперативных согласований.
